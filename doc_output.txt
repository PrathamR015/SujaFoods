1. Introduction



1.1 Purpose

This document defines the functional requirements for a MERN-stack landing page dedicated to selling freeze-dried food products. It covers product listing, individual product detail pages, the order booking flow, administrative management, and user account features.

This specification is intended for use by developers, designers, and project stakeholders to align on scope before development begins.



1.2 Scope

The system will be a web application built using:

React.js — frontend UI and routing

Node.js + Express.js — REST API and business logic

MongoDB + Mongoose — data persistence

JWT — authentication and session management



The application will allow customers to browse freeze-dried food products, view detailed product information, and place orders online. An admin panel will allow the business owner to manage products and orders.

8. Non-Functional Considerations



The following non-functional aspects should be planned for during development. These are not functional requirements but are critical for production readiness.



Area

Recommendation

Image Storage

Use Cloudinary for product image uploads. Store only the public_id and URL in MongoDB. Avoid storing binaries in the database.

Payment Gateway

Integrate Razorpay (recommended for India) or Stripe. Never handle raw card data on your server — use the gateway's hosted fields.

API Security

Apply rate limiting (express-rate-limit), CORS whitelist, Helmet.js headers, and input sanitisation (express-validator) on all public endpoints.

Performance

Index MongoDB on: category, price, isDeleted, and createdAt fields. Use React.lazy for code splitting. Serve images at multiple sizes via Cloudinary transforms.

Environment Config

All secrets (JWT secret, DB URI, Cloudinary keys, SMTP credentials) must be stored in environment variables. Never commit .env to version control.

Deployment

Frontend: Vercel or Netlify. Backend: Railway, Render, or a VPS. Database: MongoDB Atlas. Use separate environments for development and production.
